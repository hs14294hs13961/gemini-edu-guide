版本：v1.0（2026-07-09）

## 1. 專案目標（True North）

AI 功能迭代太快，初學者與老手都有認知負擔。本網站要成為一個**可直接點閱、可持續擴張**的 Gemini 教育應用學習站，整合所有既有教材（PARTS、Gems、Canvas、NotebookLM、Workspace、GAS）與未來教材。

一句話目標：**讓老師在 30 秒內找到「現在的任務該用哪個 Gemini 功能、怎麼做」，並能照著做完。**

驗收標準：

- 初學者能沿「學習路徑」從零走完一個模組
- 老手能用「任務卡」直接跳到需要的工具與提示詞
- 新增一課教材 = 新增一個內容檔案，網站自動長出頁面與篩選標籤

## 2. 目標對象

| 對象 | 需求 | 對應設計 |
|------|------|----------|
| 校內/研習教師（初學者，主要） | 循序漸進、不被術語嚇跑 | 學習路徑軸、難度標示「入門」、提示詞可一鍵複製 |
| 進階教師與行政（老手） | 快速跳關、直接找工具 | 任務卡篩選（受眾 × 難度 × 工具）、GAS/自動化專區 |
| 講師本人（Shao-Chun） | 講課時的教材集中站 | 每模組對應工作坊簡報章節，含外連原文 |

## 3. 範本網站分析

### gemini-edu-handbook.vercel.app（主要參考：配色、手冊架構）

- 結構：共同基礎 → 引導工具 → 任務實作 → 協作資源
- 五介面專區，每區走「概念學習 → 引導判斷 → 任務實作」三段節奏
- 首頁有「建議起步路徑」（新手照順序、熟手跳關）＝雙軸並行的原型
- 亮點：介面診斷器、跨介面工作流模擬器、五介面比較表
- 有「來源與版本注意事項」頁，處理 AI 產品迭代過期問題

### claude-edu-handbook.vercel.app（參考：觀念與實作卡設計）

- 任務卡設計：受眾（教師/學生/行政/…）× 難度（入門/進階/挑戰）雙重篩選
- 每張卡標注：準備資料數、預估時間、步驟、檢核方式
- 概念層（Concept）與實作層（Doing）分離

**注意**：兩站為吳奇老師作品。架構與互動模式可借鏡，**文案、程式碼、視覺素材不可複製**，需原創。

## 4. 資訊架構（雙軸並行）

```

│   篩選：受眾（教師/行政/講師）× 難度（入門/進階/挑戰）× 工具
├── 引導工具（後期）

```

## 6. 內容整合策略：站內摘要 ＋ 外連原文

- 每個模組頁放：情境引言（為什麼需要）、核心步驟、提示詞範例（一鍵複製）、檢核提醒
- 詳細圖文教學外連 Notion 原文；工具直接外連（Gems、GAS、機器人）
- 外連統一經過 `data/links.ts` 管理，方便日後檢查失效連結

## 7. 技術棧

| 項目 | 選擇 | 理由 |
|------|------|------|
| 框架 | Next.js（App Router）+ TypeScript | 與範本同路線，靜態輸出、互動元件皆可 |
| 樣式 | Tailwind CSS | 快速一致，AI 協作友善 |
| 內容 | MDX + frontmatter（`content/` 目錄） | 新增教材＝新增檔案，非工程師可讀 |
| 部署 | GitHub + Vercel 自動部署 | 免費、有預覽網址 |
| 語言 | 繁體中文（zh-TW） | 台灣教育現場用語 |

## 8. 內容模型（frontmatter schema）

```yaml
---
title: 用 PARTS 架構寫出第一個教案提示詞
module: parts            # parts | gems | canvas | deep-research | notebooklm | workspace | gas
audience: [teacher]      # teacher | admin | trainer
level: 入門              # 入門 | 進階 | 挑戰
time: 15                 # 預估分鐘
tools: [gemini-app]      # 使用到的介面/工具
prepare:                 # 準備資料清單
  - 一課的課文或單元名稱
sourceLinks:             # 外連原文
  - label: Notion 完整筆記
    url: https://...
updated: 2026-07-09
---
```

## 9. 新增教材 SOP（持續擴張的核心流程）

1. **收集**：講完課/寫完筆記後，把 PDF 或原始素材丟進 `materials/` 資料夾
2. **改寫**：請 Claude 讀素材 → 產出符合 schema 的 MDX 草稿（站內摘要＋外連）
3. **檢核**（品質 Evals，缺一退回）：
   - 提示詞範例可直接複製使用，且欄位用「[方括號]」標示需替換處
   - 有具體情境引言（老師遇到什麼困難）
   - 有「AI 會出錯」的檢核提醒（人工查證段落）
   - 難度、受眾、時間標籤已填
4. **預覽**：`npm run dev` 本機看，或推 GitHub 看 Vercel 預覽網址
5. **上線**：合併進 main，自動部署
